The RNAmute Web App Tutorial

Input Screen

The input screen of the RNAmute Web App:

The user must fill in all input fields:

  1. RNA sequence field - Insert here your RNA sequence. The sequence length should not exceed 200 bases.
  2. Number of mutations - The number of point mutations to insert into the RNA sequence. The number should be a positive number greater than or equal to 1.
  3. Start of reading frame - In case you do not want to take into account mutations that change amino acids in the corresponding protein, the “Do not change amino acids” checkbox should be checked and “Start of reading frame” should be specified. For example, if the first base in the sequence is a first base in some codon of the reading frame, the value is 1, etc.
  4. Clustering resolution for tables - Here you must specify the clustering resolution for each one of the Results Tables. The tables will be calculated according to three different distance measures: tree-edit, Hamming, and base-pair. The clustering resolution will determine how the results will be arranged in three Results Tables. Clustering resolution of X means that distances are sorted in each group and if there are two distances such that the difference between them is less than X, these distances are in the same group. The values should be positive numbers greater than 0.
  5. dist1 - This parameter is used in the algorithm and is needed for filtering suboptimal solutions that are close to the optimal. It is recommended that the value for dist1 will be about 25% of the sequence length, and this value should be lowered if more solutions are desired.
  6. dist2 - This parameter is used in the algorithm and is needed for filtering the suboptimal solutions that are close to each other. It is recommended that the value for dist2 will be about 25% of the sequence length, and this value should be lowered if more solutions are desired.
  7. e-range - Our algorithm uses the RNAsubopt routine from the Vienna RNA package [2,3] that contains this parameter. In general, a larger e value will provide better results but also a longer running time. We suggest that e-values between 8-15 will be used for sequence length of about 100 bases. It is better to use lower values first and if the running time is short, you may increase the e-range and try another run.
  8. Type of distance - Our algorithm performs some filtering of suboptimal solutions and of final results. We allow to choose one of two types of distances for filtering.
  9. Method - We provide 4 different complexity modes for our algorithm. See more information in [1]. We suggest to use the two “Fast” options that are available. The first option “Fast, only stabilizing” is very fast and provides many solutions, whereas the third option “Fast, stabilizing & destabilizing” is slower but provides many more solutions.
  10. Results - We provide two possible ways for obtaining results:
    1. Without email - After submitting the sequence, you will obtain the page with a link where you will observe the results after the algorithm will finish. You may save this link and open it afterwards. In case you open the link and the calculation did not finish yet, you will receive a corresponding notification message and you will be asked to refresh the page later.
    2. Sent by email - If you choose this option, you must specify your email address and the link with results will be sent to your email address after the algorithm will finish.

In order to start the algorithm, the “Submit Form” button should be pressed, and the results will be available by the link described in item 10 above.

Results Pages

The link with the results is either sent by email or available at the begining of the run, depending on the user's choice described in item 10 above.

See below the Results Page that is obtained as a consequence of using example values:

The Results Page contains the following information:

  1. The query RNA sequence at the top.
  2. The rightmost table with the parameters that were filled in the input screen. This table does not contain the clustering resultion, which can be viewed by inspecting the distance tables on the left.
  3. Three tables where the results are arranged according to the tree-edit, base-pair, and Hamming distances. Each row in the tables contains some distance range and the number of mutations that are within this distance range. See more information on these tables below.
  4. It is possible to download the results in the text format. Each row in the file results.txt contains: mutation name, sequence, dot-bracket representation, energy, tree-edit distance, base-pair distance, and Hamming distance.

Each table is arranged according to the corresponding clustering resolution. See also information about clustering resolution in the 4th item in the list given at the “Input Screen” section above. The average distance of all found mutations is availabe at the bottom of each table. It is possible to change the clustering resolution for each table and the table will be updated after pressing the “UPDATE” button.

It is possible to see the list of mutations that fall into some distance range of specific distance type, by clicking on the corresponding distance range. For example, clicking on the 22-26 distance range in the Hamming distance table will open the following page that lists all 19 two-point mutations that have a distance from the wild type that falls into the 22-26 Hamming distance range:

Each row in the table contains: mutation name, corresponding distance from the wild type, free energy of the secondary structure, and dot-bracket representation of the secondary structure.

It is possible to see more information about each mutation in the table by clicking on the mutation name. For example, if we press on G7C-A9U, we will get the following page:

The following information can be observed on this page:

  1. Mutation name at the top.
  2. Drawings of the secondary structures of the wild type and mutation.
  3. RNA sequences of the wild type sequence and mutation. Mutated bases in the mutated sequence are painted in red.
  4. Free energies of the wild type and mutation secondary structures.
  5. Dot-bracket representations of the secondary structures of the wild type and mutation.
  6. Tree-edit, base-pair, and Hamming distances between dot-bracket representation of the wild type and mutation.
